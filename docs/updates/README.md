# Update notes

This directory tracks progress and changes to the project in markdown files.

## How to use

- **One file per date** – Use `YYYY-MM-DD.md` for the day’s updates (e.g. `2026-01-28.md`).
- **One file per topic** – Or use descriptive names like `rspec-fixes.md`, `dhan-ws-migration.md` when you want a longer, topic-based note.
- **Short entries** – Bullet points and small sections are enough; link to code or PRs when useful.
